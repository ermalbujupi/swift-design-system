## Vision

Most SwiftUI component libraries are either too simple (just wrappers) 
or too opinionated (locked into one design language). This library aims 
to be the foundation layer‚Äîproviding well-architected, accessible 
primitives that teams can theme to match any brand.

## Principles

- **Accessible by default** ‚Äî VoiceOver, Dynamic Type, and Reduce Motion 
  supported out of the box
- **Themeable** ‚Äî Design tokens, not hardcoded values
- **Composable** ‚Äî Small primitives that combine into complex components
- **Documented** ‚Äî Every component has usage examples and API documentation
